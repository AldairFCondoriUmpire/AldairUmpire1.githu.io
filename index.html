Infografía sobre los principios, fases y mejores prácticas en el desarrollo de software

¿Qué es el Software?

El software es un conjunto de programas, instrucciones y reglas informáticas para ejecutar ciertas tareas en una computadora. Incluye procedimientos, reglas, documentación y datos asociados que forman parte de las operaciones de un sistema de computación.

Ingeniería del Software

La Ingeniería del Software se define como la aplicación práctica del conocimiento científico en el diseño y construcción de programas de computadora y la documentación asociada requerida para su desarrollo, operación y mantenimiento.

Propósito

Establecer principios y métodos de ingeniería para obtener software de manera rentable, que sea fiable y que opere en máquinas reales.

Enfoque

También conocida como "desarrollo de software" o "producción de software", busca aplicar metodologías sistemáticas para la creación de productos software de calidad.

Fase de Construcción en el Ciclo de Vida

Diseño

Transformación de requisitos en especificaciones de diseño que servirán como base para la codificación.

Codificación

Los desarrolladores escriben el código según las especificaciones del diseño, utilizando lenguajes y herramientas seleccionadas.

Pruebas

Verificación de que el software funciona correctamente y cumple con los requisitos establecidos.

Buenas Prácticas de Programación

Utilizar nombres con sentido, precisos y descriptivos para variables, funciones, parámetros, clases y métodos.
Crear funciones reducidas (idealmente no más de 20 líneas) que sigan el "principio de responsabilidad única".
Reducir el número de argumentos en funciones (idealmente menos de tres).
Evitar la duplicación de código (principio DRY).
Usar comentarios solo cuando el código no puede explicarse por sí mismo.
Mantener un formato consistente en los archivos de código.
Manejar los errores adecuadamente, separando la lógica de manejo de errores del negocio.
Mantener un único propósito para cada clase (principio de responsabilidad única).
Asegurar que las clases estén abiertas a la extensión pero cerradas a la modificación (principio abierto/cerrado).
Buscar bajo acoplamiento (poco conocimiento entre clases) y alta cohesión (independencia interna de la clase).

Roles en la Construcción de Software

Desarrolladores de Software

Crean el proyecto final mediante la programación en diversos lenguajes. Trabajan según su experiencia y especialización (front-end, back-end, full-stack).

Ingenieros de Software

Aplican principios de ingeniería para crear software y sistemas que resuelvan problemas de manera general, considerando aspectos técnicos y de gestión.
